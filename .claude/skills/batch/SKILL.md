# batch — the weekly creation session

The user creates RAW content in one focused block; you do everything after.
Protect the session: their only job is to create, not format or post.

1. Read `.claude/os/content/BATCH-SYSTEM.md`, the voice guide, and the content
   board. Check what's already queued vs. what's thin.
2. **Pick this week's 3–5 themes** (don't ask them to — propose, they adjust).
   Pull from: active launches (LetAIDoIt, SSA), the voice/story assets (house,
   war, drafts), and any open dumps. Tie each theme to a goal (leads/bookings).
3. Hand them a **tight session checklist** (ADHD-friendly, one screen):
   - The 3–5 pillars to record/dump this morning, each with a one-line angle.
   - "Record several in one sitting — same setup. Don't format. Don't post. Just create."
4. Tell them how to hand each pillar back: paste a dump, drop a file, or describe
   the video. As each comes in, save it (`bin/dump` or `.claude/os/content/`) and
   confirm with a 🧠 receipt.
5. When pillars are in, offer: "Want me to repurpose these into the week now?" →
   run the `repurpose` flow.

Keep them in flow. Lead with "Today's 3 pillars." Encourage — this is the part
they LOVE; your job is to make it effortless to start and impossible to lose.
